
# Suspended state

An `AudioContext` created without user interaction will be in the `suspended` state, in which no sound can be produced.

This is fine by itself, just make sure to resume the context once on user interaction.

```javascript
const onClick = async () => {
    await channelsInstance.resumeContext();
}
```
TLDR: The `audioContext` that is used must have been created or resumed on user interaction.

> To check whether the context is suspended: `channelsInstance.contextIsSuspended`
